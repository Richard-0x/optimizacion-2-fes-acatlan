\section{Introducción}
El método de \textbf{Ramificación y Acotamiento} es un algoritmo para resolver problemas de Programación Entera (Puro o Mixto). 


\begin{nota}{Idea}
El algoritmo se basa en resolver iterativamente el \textit{modelo relajado} (quitando la restricción de números enteros usando Simplex). Si la solución arroja decimales, se "ramifica" dividiendo el problema original en dos nuevos subproblemas excluyentes para obligar a la variable a tomar un valor entero.
\end{nota}

\section{Algoritmo}

\begin{mytheo}{Algoritmo}{pasos_ryc}
Problema de maximización:
\begin{enumerate}
    \item \textbf{Resolver el modelo relajado:} Resuelve el problema con Simplex. Si la solución es entera, Es el óptimo. Si no, ve al paso 2.
    \item \textbf{Ramificación :} Elige una variable $x_i$ que tenga valor fraccionario $v$ (ej. $x_1 = 3.75$). Crea dos subproblemas (nodos hijos):
    \begin{itemize}
        \item Subproblema 1: Agrega la restricción $x_i \le \lfloor v \rfloor$ (ej. $x_1 \le 3$).
        \item Subproblema 2: Agrega la restricción $x_i \ge \lceil v \rceil$ (ej. $x_1 \ge 4$).
    \end{itemize}
    \item \textbf{Resolver y Acotar:} Resuelve los subproblemas relajados. El valor de $Z$ de un nodo actúa como una cota superior para todas sus ramas descendentes. 
    \begin{itemize}
        \item \textbf{Infactibilidad:} Las restricciones del subproblema no tienen solución.
        \item \textbf{Integridad:} La solución del subproblema es completamente entera. Si su $Z$ es mejor que la cota actual ($Z_{cota}$), se convierte en nuestra nueva mejor solución.
        \item \textbf{Cota:} El valor óptimo fraccionario del subproblema es \textit{peor o igual} a la mejor solución entera que ya tenemos ($Z \le Z_{cota}$).
    \end{itemize}
    \item \textbf{Iteración:} Repite los pasos 2 a 4 con los nodos vivos hasta que no quede ninguno por explorar.
\end{enumerate}
\end{mytheo}

\begin{alerta}{Regla importante}
Al descender en el árbol (agregar más restricciones), el valor de la función objetivo $Z$ \textbf{nunca mejorará}. En maximización, $Z$ siempre empeorará o se mantendrá igual; esto es lo que permite podar el árbol matemáticamente.
\end{alerta}

\section{Ejemplo }

\begin{ejercicio}{Maximización de Utilidad}{ej1}
Resolver el siguiente modelo de programación entera:
\begin{align*}
    \text{Max } Z = 5x_1 &+ 4x_2 \\
    \text{s.a. } \quad x_1 + x_2 &\le 5 \\
    10x_1 + 6x_2 &\le 45 \\
    x_1, x_2 &\ge 0 \text{ y enteros}
\end{align*}

\textbf{Paso 1: Solución del modelo relajado ($P_0$)} \\
Ignorando la condición de enteros, resolvemos el sistema de ecuaciones de las restricciones activas:
$10x_1 + 6(5 - x_1) = 45 \implies 4x_1 + 30 = 45 \implies x_1 = 3.75$ \\
Sustituyendo obtenemos $x_2 = 1.25$. \\
La función objetivo es $Z = 5(3.75) + 4(1.25) = 23.75$. \\
Como $x_1$ y $x_2$ son fracciones, elegimos $x_1$ para ramificar.

\vspace{0.3cm}
\textbf{Paso 2: Ramificación ($P_1$ y $P_2$)} \\
Dado $x_1 = 3.75$, creamos dos restricciones: $x_1 \le 3$ y $x_1 \ge 4$.
\begin{itemize}
    \item \textbf{Para $\mathbf{P_1}$ (agregando $x_1 \le 3$):} 
    Al resolver, el óptimo se encuentra en $x_1 = 3$. Sustituyendo en las restricciones, $x_2 = 2$.
    $Z = 5(3) + 4(2) = 23$. ¡Es una solución entera! Nuestra \textbf{Cota Inferior} actual es $Z = 23$. Podamos esta rama por integridad.
    
    \item \textbf{Para $\mathbf{P_2}$ (agregando $x_1 \ge 4$):}
    Al resolver con $x_1 \ge 4$, el máximo se da en $x_1 = 4$.
    Sustituyendo: $10(4) + 6x_2 \le 45 \implies 6x_2 \le 5 \implies x_2 = 5/6 \approx 0.833$.
    $Z = 5(4) + 4(5/6) = 23.33$.
    Como $23.33 > 23$ (nuestra cota), existe la posibilidad matemática de encontrar una solución entera mejor en esta rama, así que debemos ramificar sobre $x_2 = 0.833$.
\end{itemize}

\textbf{Paso 3: Segunda Ramificación ($P_3$ y $P_4$)} \\
Dado $x_2 = 0.833$, creamos: $x_2 \le 0$ y $x_2 \ge 1$ heredando la restricción $x_1 \ge 4$.
\begin{itemize}
    \item \textbf{Para $\mathbf{P_3}$ (agregando $x_2 \le 0 \implies x_2 = 0$):}
    $10x_1 + 0 \le 45 \implies x_1 \le 4.5$. Para maximizar, $x_1 = 4.5$.
    $Z = 5(4.5) + 0 = 22.5$.
    Como $22.5 \le 23$ (nuestra cota), podamos esta rama por cota. No nos sirve.
    
    \item \textbf{Para $\mathbf{P_4}$ (agregando $x_2 \ge 1$):}
    Tenemos $x_1 \ge 4$ y $x_2 \ge 1$. Evaluando en la restricción principal:
    $10(4) + 6(1) = 46 \not\le 45$. El modelo es infactible. Podamos esta rama por infactibilidad.
\end{itemize}

Todos los nodos han sido evaluados o podados. La solución óptima es la mejor cota encontrada: $\mathbf{x_1 = 3, x_2 = 2, Z = 23}$.
\end{ejercicio}
